Release history

Changelog

Every update to MinkNote, from the first TestFlight build to today.

1.1 (1016)

March 3, 2026

I'm really excited about this release — there are a few workflow improvements I think you're going to love.

NEW

Improvements

  • Keyboard navigation between all 3 panes: use ⌃⌥← and ⌃⌥→ to move focus between journals, notes, and the note editor (in MD mode)
  • New note creation flow: press ⌘N, enter the note title and immediately start typing
  • Support for multiple note windows: double-click a note or press ⌘↓ and your note opens in a new window
  • Support added for Log or Journal (date-based) entries
  • Right-click a note to show Frontmatter
  • Improved support for Apple Notes exports (shows filename as title when note contains no Markdown)
  • Keyboard shortcut added for deleting notes: ⌘⌫
  • Added 2 new HowTo notes
FIX

Bug Fixes

  • Notes with duplicate filenames no longer disappear
  • Many minor fixes

1.1 (991)

February 23, 2026
NEW

Improvements

  • Double-click to open a note in a new window — supports many windows simultaneously
  • Preview mode no longer needed; links now work directly in WYSIWYG mode
FIX

Bug Fixes

  • Filenames containing periods no longer lose the file extension
  • Duplicate article UUID no longer crashes the app

1.1 (943)

February 4, 2026
NEW

Improvements

  • Improved the text formatting toolbar (see the Aa button next to Tags or use ⌘⇧Y)
  • Improved the Getting Started docs
  • Added option to add Frontmatter to a note that doesn't have it (right-click in note list)
  • Added a Note menu to the macOS status bar
  • Added ability to clear specified recent projects
  • Moved "Hide Frontmatter" option to Prefs › Files
FIX

Bug Fixes

  • Fixed issue where the Getting Started docs were not showing after launching the app for the first time
  • Fixed issue where a note's tag and favourite info might not appear in the sidebar after restarting — previously required "Rebuild Index"; now handled automatically
  • Fixed issue where the last selected note was not always reloading after changing projects

1.1 (889)

January 30, 2026
NEW

Improvements

  • Lots of options for customising your journal — right-click and choose Journal Properties in the sidebar
  • New Projects tab in Preferences with an improved UI for switching between projects
  • Double-click to expand or collapse journals and folders
  • Note selection state preserved when changing projects

1.1 (817)

January 30, 2026
NEW

Improvements

  • Note search field hidden by default
  • Edit modes simplified — use ⌘E to cycle between Markdown and WYSIWYG
  • Note identifiers now use a standard format
  • Performance improvements
  • Toggle Favourite status with ⌘⇧L
  • Double-click to toggle journal and folder expansion
  • Improved Preferences
FIX

Bug Fixes

  • Search results layout improved
  • Fixed notes scrolling to the bottom when being edited
  • Fixed data loading when switching to a new project
  • Fixed double appearance of expansion carets for sidebar sections (Sequoia)
  • Fixed jumping tag sort button
  • Fixed "last selected note" behaviour
  • Fixed journal selection when toggling "Show sample notes" (thanks JhnWyclf)
  • Ensured note list updates after note deletion from the Tags section
Tech note: Earlier versions used a different format for internal note identifiers. These have now been standardised for long-term stability. If needed, the app will prompt you to do a one-click update. This is a one-time adjustment during the beta period and does not change your notes.
